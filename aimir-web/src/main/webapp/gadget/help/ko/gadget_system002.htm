제조사 모델 관리

 Top  Previous  Next

시스템에서 사용하는 장비의 제조사, 모델 정보 및 특정 모델에 특화된 정보조회, 모델 사진 등의 서비스를 제공 합니다.

 

미니 가젯
 

현재 시스템에 등록되어 있는 장비 목록을 간단하게 확인 할 수 있습니다. 장비별, 제조사별로 정렬할 수 있습니다.

 

제조사 모델 관리 미니 버전은 아래와 같이 3개의 패널로 구성됩니다.

1. 공급사 이름

2. 장비별/제조사별 정렬 선택

3. 장비별/제조사별 정렬 선택에 따른 목록

 

 

1. 공급사 이름

공급사 이름을 명시합니다.

 

2. 장비별/제조사별 정렬 선택

3번 화면에 나타나게 될 결과에 대해 정렬 기준을 선택합니다.

장비별 정렬

집중기, 모뎀, 미터 등 타입별로 장비 목록을 보여줍니다.

제조사별 정렬

장비를 제작한 제조사별 장비 목록을 보여줍니다. 목록에서 등록된 장비의 모델명을 확인 할 수 있습니다.

 

3. 장비별/제조사별 정렬 선택에 따른 검색 결과

2번에서 선택한 정렬에 따라 장비 목록을 보여줍니다.

 

 

맥스 가젯
 

각 장비에 대한 상세정보를 확인 할 수 있습니다.

왼쪽 장비목록은 미니 가젯과 동일하게 장비별, 제조사별로 정렬 할 수 있습니다.

목록에서 제조사를 선택시 오른쪽에 제조사 상세 정보가, 장비 선택 시는 오른쪽에 해당 장비에 대한 상세 정보가 표시됩니다.

 

제조사 모델 관리 맥스 버전은 4개의필드로 구성되어 있습니다.

1. 공급사 이름

2. 장비별/제조사별 정렬 선택

3. 장비별/제조사별 정렬 선택에 따른 목록

4. 제조사 혹은 장비의 정보 및 추가

 

 

1. 공급사 이름

제조사 모델 관리 미니 버전의 공급사 이름과 내용 및 구성이 동일하다.

 

2. 장비별/제조사별 정렬 선택

장비 별로 화면을 볼건지 제조사별로 화면을 볼것인지를 선택하면 3번필드에 목록 내용이 나타납니다.

 

3. 장비별/제조사별 정렬 선택에 따른 목록

장비별 정렬

집중기, 모뎀, 미터 등 타입별로 장비 목록을 보여줍니다.

제조사별 정렬

장비를 제작한 제조사별 장비 목록을 보여줍니다. 목록에서 등록된 장비의 모델명을 확인 할 수 있습니다.

 

4. 제조사 혹은 장비의 정보 및 추가

•제조사 추가

제조사 추가 버튼을 누르면 새로운 제조사를 등록할 수 있습니다.

제조사명, 고유코드, 주소, 설명을 입력할 수 있습니다.

※고유코드는 별도의 지정된 코드를 입력해야 합니다.  자세한 사항은 누리텔레콤에 문의해 주시기 바랍니다.

 

•장비 추가

- Meter Program

미터 혹은 모뎀에 등록할 값을 저장해 해당하는 미터 혹은 모뎀에 적용할 수 있습니다.

등록한 Meter Program 정보는 Task Management가젯의 스케줄러나 Meter Management 가젯의 Command 파트에서 해당하는 셋팅목록 버튼을 클릭해서 실행할 수 있습니다.

 

Meter Program 은 3번 필드에서 모뎀 혹은 미터를 선택 후 등록이 가능합니다.

Meter Program 탭의 패널은 3가지로 구분됩니다.

5. Meter Program 목록

6. Meter Program Setting 내용

7. Meter Program 로그

5. Meter Progrma 목록

모뎀 혹은 미터에 셋팅할 값을 목록을 선택할 수 있습니다.

- Display Item Setting

- TOU Calendar

두 개의 Meter Program Setting을 지원하며 미터에 따라 적용내용은 달라질 수있습니다.

 

6. Meter Program Setting 내용

5번에서 선택한 목록값에 해당하는 설정 내용을 입력합니다.

설정 내용은 목록 값에 따라 패턴과 내용이 달라집니다.

 

7. Meter Program Log

3번필드에서 선택한 장비들에 6번 내용을 적용한 로그를 보여줍니다.

선택한 장비들에 6번에서 설정한 내용의 성공 개수/ 실패 개수, 시도 개수, 최종 변경일을 보여줍니다.

미터프로그램의 설정 내용이 지정한 미터에 스케줄 관리 가젯을 통해 스케줄러를 돌려 성공적으로 적용될 경우 더이상 스케줄러는 성공한 미터에 해당내용을 적용하려는 시도를 하지 않습니다.

단, 새로운 설정 내용이 업데이트되게 되면 다시 스케줄러를 돌려 설정내용이 적용될때 까지 작업을 시도하게 됩니다.
