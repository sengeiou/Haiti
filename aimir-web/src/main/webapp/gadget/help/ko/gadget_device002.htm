모뎀관리

 Top  Previous  Next

모뎀 종류별, 지역별, 통신 상태 별 모뎀 목록 관리 및 장비 추가, 삭제 기능 제공합니다.

특정 모뎀에 대한 LP log, Event Log, Unit Scanning, Monitoring, Firmware Update 기능 제공합니다.

 

미니가젯

 

모뎀관리 미니 버전은 아래와 같이 2개의 패널로 구성되어 있습니다.

1. 모뎀의 타입 및 통신상태별 차트

2. 모뎀의 타입 및 통신상태별 그리드

 

 

모뎀 타입별(ACD, Converter, HMU, IEIU, IHD, MMIU, PLC, ZBRepeater, ZEUMBus, ZEUPLS, ZMU, ZRU 등)

/통신 상태별(24시간 이내 통신 정상, 24시간 이상 통신 장애,  48시간 이상 통신 장애 등)으로 차트와 그리드를 제공합니다.

 

맥스 가젯

 

모뎀관리 맥스 버전은 아래와 같이 4개의 패널로 구성되어 있습니다.

1. 모뎀 타입, 상태, 버전, 지역 등 조건 입력

2. 모뎀이 연결된 집중기 차트 및 리스트

3. 모뎀 목록

4. 모뎀에 대한 상세 정보
 

1.   조회 및 검색

 

1. 모뎀 타입, 상태, 버전, 지역 등 조건 입력

- 모뎀 타입, 모뎀 아이디, 상태, 설치 상태, 집중기 타입, 집중기 아이디, 소프트웨어 버전, 하드웨어 버전, 지역, 설치일시, 최종통신 일시 등의 조건으로 모뎀을 검색할 수 있습니다.

- 재입력 버튼을 클릭시 조회 조건이 초기회 됩닌다.

 

2. 모뎀이 연결된 집중기의 차트 및 리스트

해당 집중기에 연결되어진 모뎀의 통신상태를 차트 및 그리드로 나타냅니다.

2번 패널의 그리드에서 집중기 하나를 선택 하면 3번 필드에 해당 집중기와 연결된 모뎀들의 목록을 출력합니다.

 

3. 모뎀 목록

•모뎀 아이디, 모뎀 타입, 집중기 아이디, 모델, SW/HW 버전, 최종통신일시 의 정보를 보여줍니다.
•통신 상태별,최종통신 올림/내림 차순, 설치일시 올림/내림 차순 별로 필터링이 가능합니다.
•조회된 모뎀 목록을 엑셀로 출력 가능합니다.

 

4. 모뎀에 대한 상세 정보

선택한 모뎀의 기본정보와 선택한 모뎀의 위치, 스케줄, 설정 정보 및 실행할 수 있는 명령등을 표시합니다.

상세정보는 4개의 탭으로 구성되어 있습니다.

•일반
•위치정보
•스케줄
•설정 정보

 

•일반

 

  일반 탭은 아래와 같이 3개의 패널로 구성되어 있습니다.

  4-1. 기본 정보

  4-2. 미터 연결

  4-3. 명령

 

 

  4-1. 기본 정보

  모뎀 아이디, 모뎀타입, SW/HW/FW 버전, 프로토콜, 제조사, 모델, 지역, 설치일시, 최종 연결 시각, 채널아이디 등 모뎀의 정보를 확인 할수 있고

  또한 연관 되어 있는 집중기 아이디 정보를 확인 할수 있습니다.
  그리고 등록,수정,삭제 버튼을 사용하여 정보 수정및 삭제를 할수 있습니다.

•등록

   a) 4-1번 패널에 있는 화면에서 [등록]버튼을 누르면 아래 화면으로 전환됩니다.

 b) 새로운 모뎀을 등록하는데 필요한 정보들을 입력한후 [등록]버튼을 클릭합니다.

모뎀아이디

모뎀의 아이디. ※필수 입력값입니다.

모뎀 타입

모뎀의 타입. ※필수 입력값입니다.

종류 : ACD, Converter, HMU, IEIU, IHD, MMIU, PLCIU, Unknown, ZBReapeater, ZEUMBus, ZEUPLS, ZMU, ZRU .

집중기 아이디

모뎀과 연결되어 있는 DCU의 ID.

하드웨어 버전

하드웨어 버전.

소프트웨어 버전

소프트웨어 버전.

프로토콜

통신프로토콜의 종류 선택. ※필수 입력값입니다.

종류 : Bluetooth, DCMA, GPRS, GSM, LAN, PLC, PSTN, Serial, SMS, WiMAX,  ZigBee

제조사

모뎀의 제조사 이름. ※필수 입력값입니다.

모델

모뎀의 모델이름. ※필수 입력값입니다.

설치일시

사용자가 입력하지 않아도 모뎀을 추가하고 나면 시스템에서 자동으로 입력됩니다.

 

•수정

   a) 4-1번 패널에 있는 화면에서 [수정] 버튼을 누르면 수정 화면으로 전환됩니다.

•삭제

   a) 4-1번 패널에 있는 화면에서 [삭제] 버튼을 누르면 모뎀이 삭제됩니다.

 

  4-2. 미터 연결

  모뎀과 연결할 미터를 선택한 후 추가 버튼을 누르면 모뎀과 미터가 연결됩니다.

※모뎀 타입에 따라 모뎀과 미터간의 관계가 일대일 이나 일대다 관계가 될 수 있습니다.

•등록

a) 미터와 연결하고자 하는 모뎀을 선택한 후 4-2번 패널에서 Add 버튼을 클릭하면 해당 화면이 아래 그림처럼 전환됩니다.

     b) 여기서 선택한 모뎀과 연결하고자하는 미터를 찾아 체크하고 OK 버튼을 누르면 미터가 연결됩니다.

 

•삭제

a) 모뎀을 선택한 후 해당 모뎀과 연결되어 있는 미터 중 삭제하고자 하는 미터를 체크한 후 Delete 버튼을 클릭하면 미터와의 연결이 끊어집니다.

 

  4-3. 명령

Command 프로토콜을 이용하여 실시간으로 모뎀의 정보를 확인할수 있는 기능을 제공합니다.

제공되는 정보로써는 모뎀에 저장되어 있는 로그를 확인할수 있는 LP Log, 이벤트 이력 버튼과,설정 세부 사항을 확인할수 있는 유닛 스캐닝 버튼, 상태 정보를 지속적으로 확인할수 있는 상태 감시 그리고 원격으로 펌웨어를 업데이트 할 수 있는 Firmware Update 버튼을 제공하여 기능을 수행합니다.

 

•Firmware Update 버튼

프로토콜이 SMS일 경우와 그렇지 않은 경우 2가지로 분류됩니다.

 

-프로토콜 타입이 SMS일 경우

Firmware Update 버튼 클릭 시 아래와 같은 업로드 창이 뜨면 ebl 타입의 펌웨어 파일을 선택하면 됩니다.

 

-프로토콜 타입이 SMS가 아닐 경우

Firmware Update 버튼 클릭 시 아래와 같이 팝업창이 뜨면 밑에 표를 참고해서 해당 값을 적고 OK버튼을 눌러주면 업데이트가 실행됩니다.

 

펌웨어 업데이트 버튼 클릭 시 위와 같은 창이 나타나며 Source Version Infromaion 부분에는 현재 집중기에 등록되어 있는 정보를, Target Version Information 에는 업데이트할 펌웨어의 정보를 입력하면 됩니다.

OTA Infomaion에는 펌웨어 업데이트에 필요한 설정들을 해당 모뎀에 맞게 입력하면 펌웨어를 업데이트 할 수 있습니다.

 

Source Version Infomaion

Source H/W Ver

업데이트 하고자 하는 모뎀의 현재 H/W 버전을 적습니다. 예) 1.0

Source F/W Ver

업데이트 하고자 하는 모뎀의 현재 F/W 버전을 적습니다. 예) 3.0

Source F/W Build

 업데이트 하고자 하는 모뎀의 현재 Build를 적습니다. 예) 77

Source BinaryFileName

업데이트 하고자 하는 모뎀의 기존 파일명을 적습니다.

 이때 "파일명.확장자 " 의 형식으로 적어야 합니다.. 예) SourceBinaryFile.ebl

Future Version Infomation

Future H/W Ver

업데이트 할 새로운 펌웨어 파일의 H/W 버전을 적습니다.

Future F/W Ver

업데이트 할 새로운 펌웨어 파일의 F/W 버전을 적습니다.

Future F/W Build

업데이트 할 새로운 펌웨어 파일의 Build를 적습니다.

OTA Infomation

Install Type

펌웨어 설치방식을 의미합니다. (default : 0)

0

Auto, 같은 버전일 경우 설치하지 않습니다.(현재 버전이 New Version과 동일하면 설치 안함)

1

Reinstall, 무조건 재설치, 설치 테스트 및 다시 내릴 경우 ,버전이 같아도 내려집니다.

2

Match, Source H/W, Source F/W, Build Version이 모두 동일하면 New Version으로 설치합니다.

3

Force, Reinstall 처럼 무조건 재설치를 하고 File도 무조건 Base file을 다운 받습니다.

Transfer Type

전송을 Unicast로 할 것인지, Multicast로 할 것인지를 결정합니다. (default : 1)

0

 Auto, 50대 기준으로 Multicat/Unicast가 자동 결정됨

1

Multicast (펌웨어 전송만 Multicast, Verify, Install은 Unicast로 진행)

2

Unicast

OTA Step

Firmware Upgade의 진행 단계를 나타냅니다. (default : 0x1f)

0x01

센서 정보 수집 및 버전 확인 (필수)

0x02

파일 전송(Data Send)

0x04

전송 이미지 확인 (Verify)

0x08

설치(Install)

0x10

업그레이드된 버전 확인(Scan)

0x1f

ALL

OTA Thread Count

전송 이미지 확인(Verify), 설치(Install) 시의 최대 Thread 개수를 의미합니다. (default : 1)

1~2 중에 선택해야 하며  "1" 을 선택시 전송 이미지 확인 후 설치, "2" 를 선택시 전송 이미지 확인과 설치를 동시에 진행합니다.

Max Retry Count

개별 센서에 대하여 최대 재시도 횟수를 지정합니다.(default:3)

Multicast Write Count

Multicast 일때, 펌웨어를 Multicast로 전송시 같은 프레임을 몇 번 전송할것인지 결정합니다.

멀티 캐스트가 아닌경우 이 값은 (default : 1) 로 지정합니다.

Do you have a diff file?

diff 파일이 존재할 경우에만 Yes를 선택합니다.

diff 파일명은" Target File의 파일명_FROM_Source File의 파일명" 형식으로 되어있습니다.

이때 diff 파일은 Target Version 파일이 있는 폴더에 위치해야 합니다.

 

•위치정보

선택한 모뎀의 위치를 입력한 위치정보에 따라 구글맵을 사용하여 표현합니다.

또한 위치 정보를 입력하고 수정 버튼을 사용하여 수정할 수 있습니다.

 

•스케줄

스케쥴 탭에서 설정할수 있는 정보로써 송신 주기,Alarm Flag,LP Choice 와 검침 시간을 수정 할수 있습니다.

 

•설정 정보

  MMIU 타입의 모뎀의 경우에만 설정 정보 화면이 나타납니다.

  phone number, Comm Speed, GPRS AT Command, Sending Flag, Meter Vendor & Model, SNTP Server IP, Upload Server IP & Port, APN ID & Password, SMS Service Center Address를 설정할 수 있습니다.

 

 

phone number : 모뎀의 전화번호

Comm Speed : 통신 속도, 1200,2400,4800,9600,19200,38400,115200 7가지 속도를 선택할 수 있습니다.

GPRS AT Command : SMS 보낼때 명령어를 설정합니다. ATD*99***1#, ATD*99# 두가지 command중 선택할 수 있습니다.

Meter Vendor & Model : 미터의 제조사와 모델을 설정합니다.

SNTP Server IP : NTP서버의 IP를 입력합니다.

Upload Server IP & Port : Upload한 데이터를 받을 서버의 IP와 Port를 입력합니다.

APN ID & Password : APN(Access Point Name)에 접근하기 위한 ID와 Password를 입력합니다.

SMS Service Center Address : 모뎀이 SMS보낼때 그기능을 서비스하는 센터의 주소를 입력합니다.
