명령이력관리

 Top  Previous  Next

장비에 내린 명령(Operation)들의 목록과 각각의 시도 횟수, 성공률 등을 차트로 제공합니다.

장비 별로 시스템에서 제어할 수 있는 명령들에 대한 설정 및 시스템에서 지시한 명령 수행 이력들에 대한 정보를 제공합니다.

 

미니가젯

 

 

미니버전에서는 한 달을 기준으로 실행했었던 Relay Switch, Unit Scanning, Event Log, LP Log, Time Synchronization, Status Monitoring 등 다양한 명령(Operation)들의 목록과 각각의 시도횟수, 성공률 등을 표시합니다.

•[전체건수] 컬럼

전체건수에 표시되는 숫자는 전체건수(성공횟수/실패횟수) 를 의미합니다.

 

맥스가젯

 

명령이력관리 맥스 버전은 아래와 같이 4개의 패널로 구성되어 있습니다.

1. 시간별, 기간, 주별, 월별(기간)로 조건에 따라 데이터 검색 및 설정 버튼 위치

2. 조회 조건에 따른 명령 성공여부 차트

3. 조회 조건에 따른 명령 중심 그리드

4. 조회 조건에 따른 명령 성공여부에 대한 상세 그리드

 

 

1. 시간별, 기간, 주별, 월별(기간)로 조건에 따라 데이터 검색 및 설정 버튼 위치

•시간별, 기간, 주별, 월별(기간)로 다양한 조건에 따라 데이터를 검색할 수 있습니다.
•[설정버튼]

 

- 명령 탭을 클릭하면 명령 목록이 트리 형태로 표현되고 해당 명령을 클릭하면 우측 화면에 해당 명령을 가지고 있는 장비들이 나타납니다.

- 장비 탭을 클릭하면 장비 리스트가 표현되고 해당 장비를 클릭하면 해당 장비가 지니고 있는 명령이 표시됩니다.

공사별로 각각의 Operation Log Setting을 지닐 수 있고, 공사별 Operator가 Operation Log Setting의 변경 권한을 가집니다.

(※변경 권한을 지니지 않은 사용자는 조회만 가능하고 변경은 불가능합니다.)

 

- 유저 그룹 관리에서 enable한 Operation 들에 대해서만 로그를 Enable/Disable 할 수 있도록 합니다.

 

2. 조회 조건에 따른 명령 성공여부 차트

조회된 조건에 따라 검색된 로그들을 성공/실패에 따라 횟수를 차트로 보여줍니다.

차트 클릭 시 디테일한 정보가 4번 패널에 표시됩니다.

 

3. 조회 조건에 따른 명령 중심 그리드

명령어를 중심으로 해당 명령이 몇번 시행되었고 몇번의 성공과 실패가 있었는지를 표시해줍니다.

순위

Comand에 따라 많이 사용한 순위를 의미합니다.

명령

실행한 Command의 이름을 표시합니다.

전체 건수

전체 횟수(성공횟수/실패횟수) 를 의미합니다.

 

4. 조회 조건에 따른 명령 성공여부에 대한 상세 그리드

•조회 조건에 성공/실패여부를 설정하지 않거나 2번 패널의 차트에서 성공/실패 조건을 선택하지 않을 시에는 전체 데이터를 가져오는 것을 기본으로 합니다.
• 엑셀 출력 기능을 제공합니다.

발생 시각

명령을 실행한 시간을 의미합니다.

대상 타입

- 명령을 실행한 장비의 타입을 의미합니다.

- 대상타입의 종류는 하단의 참고합니다.

대상

명령을 실행한 장비의 아이디를 의미합니다.

수행자 타입

명령을 실행시킨 사용자의 타입을 의미합니다.

수행자

명령을 실행시킨 사용자의 아이디를 표시합니다.

명령

장비에 실행시킨 명령을 의미합니다.

상태

- 실행시킨 명령의 성공/ 실패 여부를 의미합니다.

- 실패했을 경우 에러 메세지가 표시됩니다.
