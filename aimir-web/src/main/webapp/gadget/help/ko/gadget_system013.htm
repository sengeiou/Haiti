3. 스케줄 등록, 수정, 삭제 및 시작/멈춤 기능

•1번 패널에서 선택한 작업에 스케줄을 추가하거나 수정, 삭제할 수 있습니다.
•simple expresstion에서 화살표를 클릭하면 선택한 해당 스케줄을 바로 한번 실행시킵니다.
•2번 패널에 나타난 스케줄 중에 하나를 선택 후 시작/정지 버튼을 통해  일시적으로 cron expresstion으로 동작되는 스케줄을 Paused 상태로 만들 수 있습니다.

 

•스케줄 추가하는 법:

a) Add 버튼을 클릭합니다.

b) Schedule name을 입력하고 스케줄을 돌릴 그룹 타입을 선택합니다. 이때 그룹 정보는 그룹관리 가젯에 설정된 그룹 정보를 바탕으로 합니다.

c) simple expresstion, cron 표현 중 어떤 형식으로 스케줄을 실행시킬지 선택합니다.

   - simple expresstion을 선택했을 경우 반복간격옆의 빈 칸에 얼마간의 기간을 두고 반복시킬지 시간을 적어줍니다. 단위는 millisecond 입니다.

   빈 칸에 1000을 적을 경우 1초에 한번씩 해당 스케줄을 실행하게 됩니다.

   repeat interval 옆의 화살표를 클릭하면 simple expression에 입력한 내용대로 스케줄을 실행합니다.

   - cron expresstion을 선택했을 경우에는 초, 분, 시간, 일, 월, 요일 별로 simple 보다는 긴 기간을 두어 반복적으로 스케줄을 실행하도록 합니다.

     만약 초, 분, 시간, 일, 월, 요일 이라고 적혀있는 빈칸에 각각 0, 0, 12, *, *, ? 이라고 적는다면 매일 12 : 00pm 에 스케줄이 돌아가게 됩니다.

     cron 표현 선택시 작성법은 버튼을 클릭 해 참고하도록 합니다.

 

•스케줄 삭제하는 법:

a) 삭제하고자 하는 스케줄을 2번 패널에서 선택합니다.

b) 3번패널에 보이는 삭제버튼을 클릭합니다.

 

•스케줄 수정하는 법:

a) 수정하고자 하는 스케줄을 2번 패널에서 선택합니다.

b) 선택한 스케줄의 내용이 3번패널에 나타나면 원하는 내용으로 수정한 후 수정버튼을 누릅니다.

 

•스케줄 PAUSED 상태로 만들기

a) PAUSED 상태로 만들고자 하는 스케줄을 선택한 후 3번패널의 Stop버튼을 누릅니다.

b) 2번 패널에서 선택한 스케줄의 상태가 PAUSED로 변하고 정지버튼이 시작버튼으로 변하면 성공.

 

•PAUSED 상태인 스케줄 NORMAL로 만들기

a) PASUED 상태인 스케줄을 선택한 후 3번패널의 시작버튼을 누릅니다.

b) 2번 패널에서 선택한 스케줄의 상태가 NORMAL로 변하고 시작버튼이 정지버튼으로 변하면 성공.

 

4. 스케줄 로그

선택한 작업에 대한 스케줄이 돌면서 발생한 로그를 표시합니다.

날짜별, 3번패널에서 등록한 스케줄 이름별, 결과 별로 검색이 가능합니다.

Date

스케줄이 실행된 날짜

Job Name

실행된 작업의 이름

Schedule Name

실행된 스케줄의 이름

Result

All, SUCCESS, FAIL, INVALID_PARAMETER, COMMUNICATION_FAIL 네가지 항목이 표시됩니다.

Message

에러 메세지 혹은 어떤 미터가 실패했는지 성공했는지 여부를 표시합니다.
