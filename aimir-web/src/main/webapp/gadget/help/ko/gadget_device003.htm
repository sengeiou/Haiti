미터관리

 Top  Previous  Next

전기,가스, 수도, 열량 미터 등 공사가 공하는 에너지원 목록 및 상세 정보 연관된 장치 정보, 미터 별 검침데이터 ,위치 정보, 미터 설치 정보 사진 등을 제공

 

미니가젯

 

미터관리 미니 버전은 아래와 같이 2개의 패널로 구성되어 있습니다.

1. 미터의 타입 및 통신상태별 차트

2. 미터의 타입 및 통신상태별 그리드
 

 

미터 타입별(EnergyMeter, GasMeter, HeatMeter, SolarPowerMeter, VolumeCorrector, WaterMeter 등)

/통신 상태별(24시간 이내 통신 정상, 24시간 이상 통신 장애,  48시간 이상 통신 장애 등)으로 차트와 그리드를 제공합니다.

 

맥스가젯

 

미터관리 맥스 버전은 아래와 같이 4개의 패널로 구성되어 있습니다.

1. 미터 타입, 상태, 설치일시, 지역 등 조건 입력

2. 미터가 연결된 집중기 차트 및 리스트

3. 미터 목록

4. 미터에 대한 상세 정보

 

 

1. 미터 타입, 상태, 설치일시, 지역 등 조건 입력

- 미터 타입, 미터 아이디, 미터 그룹, 상태, 지역, 집중기 아이디, 제조사, 모델, 모뎀 모델, 최종통신일시, 설치아이디, 상태, Install Property, 등의 조건으로 미터를 검색할 수 있습니다.

- 재입력 버튼을 클릭시 조회 조건이 초기화 됩닌다.

 

2. 미터가 연결된 집중기의 차트 및 리스트

해당 집중기에 연결되어진 미터의 통신상태를 차트 및 그리드로 나타냅니다.

2번 패널의 그리드에서 집중기 하나를 선택 하면 3번 필드에 해당 집중기와 연결된 미터들의 목록을 표시합니다.

 

3. 미터 목록

•미터 아이디, 미터 타입, 계약 번호, 고객 번호, 고객 명, 집중기 아이디, 제조사, 모델, SW/HW 버전, 임계치, 지역, 설치일시의 정보를 보여줍니다.
•통신 상태별,최종통신 올림/내림 차순, 설치일시 올림/내림 차순 별로 필터링이 가능합니다.
•조회된 미터 목록을 엑셀로 출력 가능합니다.

 

4. 미터에 대한 상세 정보

선택한 미터의 기본정보와 선택한 미터의 위치, 검침 데이터 및 실행할 수 있는 명령등을 표시합니다.

상세정보는 4개의 탭으로 구성되어 있습니다.

•일반
•위치정보
•검침데이터

  4-1. 장비 정보

  또한 연관 되어 있는 집중기 아이디 정보를 확인 할수 있습니다.

모델

 그리고 등록,수정,삭제 버튼을 사용하여 정보 수정 및 삭제를 할수 있습니다.

•등록

   a) 4-1 패널에서 [Registration] 버튼을 클릭하면 아래 화면으로 전환됩니다.

 

    b) 새로운 미터를 등록하는데 필요한 값들을 입력하고 하단의 [등록] 버튼을 클릭해 미터를 등록합니다.

         - 빨간 별표(*)로 표시되어져 있는 항목은 필수 값으로 반드시 입력해야 하는 항목입니다.

         - 미터 아이디의 경우 값을 입력후 옆에 [중복확인] 버튼을 클릭해 중복되는 아이디를 갖는 미터가 없도록 확인해야합니다.

           미터아이디가 중복일 경우 미터가 등록되지 않습니다.

         - 설치일시의 경우 사용자가 입력하지 않아도 미터를 등록하고 나면 시스템에서 자동으로 등록됩니다.

•Firmware Update 버튼 클릭 시

SX2 미터를 선택한 후 Firmware Update 버튼을 누르면 아래와 같이 파일 업로드 창이 나옵니다.

파일 업로드 창에서 해당 펌웨어 파일을 선택하고 열기 버튼을 누르면 기본 설정 값으로 펌웨어 업데이트가 진행됩니다.

 

 

 

기본 설정 값

Install Type

펌웨어 설치방식을 의미합니다.    (현재 자동설정 값 : 1)

0

Auto : 같은 버전일 경우 설치하지 않습니다.(현재 버전이 New Version과 동일하면 설치 안함)

1

Reinstall : 무조건 재설치, 설치 테스트 및 다시 내릴 경우 ,버전이 같아도 내려집니다.

2

Match, Source H/W, Source F/W, Build Version이 모두 동일하면 New Version으로 설치합니다.

3

Force, Reinstall 처럼 무조건 재설치를 하고 File도 무조건 Base file을 다운 받습니다.

Transfer Type

전송을 Unicast로 할 것인지, Multicast로 할 것인지를 결정합니다.  (현재 자동설정 값 : 0)

0

Auto, 50대 기준으로 Multicat/Unicast가 자동 결정됨

1

Multicast (펌웨어 전송만 Multicast, Verify, Install은 Unicast로 진행)

2

Unicast

OTA Step

Firmware Upgade의 진행 단계를 나타냅니다. (현재 자동설정 값 : 0x1f)

0x01

센서 정보 수집 및 버전 확인 (필수)

0x02

파일 전송(Data Send)

0x04

전송 이미지 확인 (Verify)

0x08

설치(Install)

0x10

업그레이드된 버전 확인(Scan)

0x1f

ALL

OTA Thread Count

전송 이미지 확인(Verify), 설치(Install) 시의 최대 Thread 개수를 의미합니다. (현재 자동설정 값 : 1)

1~2 중에 선택해야 하며 "1" 을 선택시 전송 이미지 확인 후 설치, "2" 를 선택시 전송 이미지 확인과 설치를 동시에 진행합니다.

Max Retry Count

개별 센서에 대하여 최대 재시도 횟수를 지정합니다.(현재 자동설정 값:3)

Multicast Write Count

Multicast 일때, 펌웨어를 Multicast로 전송시 같은 프레임을 몇 번 전송할것인지 결정합니다. (현재 자동설정 값:1)
