미터와 서버시간과의 오차 정보, 미터 타임 설정 이력, 미터 타임 설정에 대한 임계치 설정 등의 서비스 제공합니다.

 

미니가젯
 

일반 조회에서는 미터와 서버시간과의 오차 정보, 미터 타임 설정 이력등의 서비스를 제공합니다.

미터와 서버 시간과의 오차 정보에서는 12시간 단위의 시간 차를 챠트로써 표현하고 미터의 시간을 동기화 하기 위한 미터 시간 동기화 로그를 통해 미터 동기화 정보를 나타냅니다.

 
맥스가젯

 

미터 시간관리 맥스버전은 3개의 탭으로 구성되어 있습니다.

•[미터 시간 오차 목록] 탭
•[미터 시간 동기화 로그] 탭
•[집중기 미터시간 임계치 설정] 탭
 
•[미터 시간 오차 목록] 탭

미터 시간차를 구간별로 그래프로 표시해줍니다.

일치, 1~12시간, 12~24시간, 24시간 이상은 임의로 나눠 놓은 구간으로 추후에 사이트 별로 의미 있는 구간으로 변경될 수 있습니다

•.TimeSync : 그리드에서 시간동기화를 할 미터목록을 선택하고 "TimeSync" 버튼을 클릭하면 선택된 미터에 대해 시간동기화를 실행합니다.

 

•[미터 시간 동기화 로그] 탭

미터 시간 동기화 로그 탭의 리스트에서는 미터 시간 동기화가 가능한 계량기 목록의 타임 시간 차를 목록으로 표현합니다.

•x축: 오늘 이전 7일간의 날짜

y축 : 각 날짜에 실행한 전체 타임 싱크 커맨드를 Status(Success, Failure, Invalid Argument, Communication Fail)에 따라서 개수를 구하여 날짜 별로 시도한 총 타임 싱크 수와 성공 실패 여부를 직관적으로 표현합니다.

•좌측에는 시간차 구간별 그래프, 우측에는 Compliance여부에 따른 그래프를 그려줍니다.
•Compliance(30일 이내에 시간 동기화를 한적이 있는 미터를 Compliance한 미터로 정의함)
•Time Diff.나 Compliance 검색 조건이 존재하는 경우 그래프는 그냥 해당하는 조건에 대해서 그립니다/

Ex) Time Diff가 12~24로 선택된 경우, 원형 그래프에는 Time Diff가 12~24인 전체 미터 개수를 표시

•Compliance가 Yes 인 경우 차트에는 Compliance가 Yes인 전체 개수를 표시

 

•Method가 All인 경우는 Auto- 날짜 별 시간차의 백분율 그래프, Manual- 날짜 별 Status 상태 그래프를 보여줍니다.
•Method가 Auto인 경우 (I210, kv2c 검침 데이터에 포함되어 있는 시간 동기화 로그)

좌측에는 요일 별 시간차 그래프를 우측에는 미터를 MCU 별로 그룹핑해서 시간 차이가 가장 많이 나는 베스트 10개의 MCU를 그래프로 표시하고, 각각의 MCU에서 각각의 시간차에 속하는 미터의 개수를 그래프로 표현합니다.

•Method가 Command 인 경우(유저가 UI상에서 타임 동기화를 실행)

좌측에는 동기화의 Status별 그래프를우측에는 해당 검색 기간 중 Status의 백분율을 표현합니다.
