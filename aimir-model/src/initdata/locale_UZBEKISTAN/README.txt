* 기초데이터 입력 규칙

1. 데이터 간의 의존성에 따라 (들어가야하는 규칙에 따라) 입력 순서를 결정한다.

2. 파일명 제일 앞의 "번호-"은 DB에 입력하는 순서를 정하는 것이며, 내림차순에 의해 정의하도록 한다.

3. 기초데이터는 많은 데이터를 넣는 작업은 아니므로, 가능한 입력속도가 빠른 xml 형식을 권장한다.

4. 데이터의 크기가 클 경우 작은 단위로 나누어 작성하도록 한다. (한번에 입력을 시도할 경우 오류를 찾아내기 힘듬)

5. 동일한 기능에 내용이 다를 경우 파일명을 동일하게 하고 뒤쪽에 구분자로 구분하여준다.
